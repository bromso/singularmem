# TypeScript SDK — Search + Retrieve Bindings (5b) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the `singularmem-node` napi binding with `Store.search()`, `Store.retrieve()`, and a frozen `adapters.{plain,claude,openai,gemini}` namespace whose entries each format a `RetrievedContext` into a provider-specific string.

**Architecture:** Build on 5a's established patterns. Two new Task structs (`SearchTask`, `RetrieveTask`) follow the existing 5-method Task pattern in `src/store.rs`. Four `#[napi]` adapter classes go in a new `src/adapters.rs` module — sync `format()` methods (no Task wrapper, since formatting is microsecond-scale CPU work). The JS wrapper class in `scripts/patch-index.js` grows two more methods that lift nested `Item` instances via `liftItem`. The `adapters` JS export is constructed at module load time and frozen. The binding crate gains 5 new dependencies (singularmem-search, singularmem-retrieve, and the three cloud-adapter crates).

**Tech Stack:** Unchanged from 5a — napi 2 + napi-derive 2, tokio rt-multi-thread for libuv Task pattern, `@napi-rs/cli` for codegen, Node 20.12+, TypeScript 5.4 for type smoke test. The new transitive deps (Tantivy 0.22.1, USearch 2.15.3, fastembed 4.4.0) are already in the workspace via `singularmem-search`.

**Spec:** `docs/superpowers/specs/2026-05-19-ts-sdk-5b-design.md`

## Known API verifications needed (read before starting Task 1)

The plan was drafted from the spec. Subsequent inspection of the actual Rust source surfaced several real differences from the type signatures the spec assumed. The implementer should treat the code blocks below as *intent* rather than verbatim — adapt field names and method signatures as needed, but preserve the JS contract (the `.d.ts` shape and `.code` strings).

**`singularmem_search::HybridHit` actual fields** (per `crates/singularmem-search/src/hybrid_query.rs`):
- `id: ItemId` (NOT `item_id`)
- `score: f32`
- `score_kind: ScoreKind` (NOT `kind`)
- `lexical_rank: Option<usize>` and `semantic_rank: Option<usize>` (NOT `bm25_score`/`cosine_score`)
- `snippet: Option<String>`

**Implication:** the napi `SearchHit`'s `bm25?: number` and `cosine?: number` from the spec **do not exist as scores in `HybridHit`**. The closest available fields are `lexical_rank` and `semantic_rank` (1-based rank positions, not scores). The implementer should rename the napi `SearchHit` fields to `lexicalRank?: number` and `semanticRank?: number` (matching the Rust shape) and update the spec's accepted-deviations note plus the type smoke test accordingly. The `kind === 'rrf'` → "ranks are populated" relationship still holds — single-ranker modes leave them as `None`.

**`singularmem_search::HybridSearchOptions` actual fields:**
- `limit: usize`, `fetch_multiplier: usize`, `rrf_k: usize`, `include_snippets: bool`
- NO `mode` field — the mode is implicit in which `HybridSearcher` constructor was used (`new` / `lexical_only` / `semantic_only`)

**Implication:** the napi `SearchOptions { mode? }` from the spec needs separate handling. The handler converts the mode string → which constructor to call, and passes `HybridSearchOptions` (without `mode`) to `search()`.

**`SearchMode` enum lives in `src/main.rs` (the root CLI), not `singularmem-search`.**

**Implication:** the napi crate needs to define its own local mode enum (or just match on the string directly). The spec's TypeScript `SearchMode` type stays as the string-literal union; the Rust binding side maps it to the choice of `HybridSearcher` constructor.

**`HybridSearchResults` actual fields:**
- `hits`, `elapsed: Duration`, `total_fused: usize`, `lexical_hits: Option<u64>`, `semantic_hits: Option<u64>`
- NO `query` field

**Implication:** `Store::search`'s napi handler must echo the query separately (the JS-side `SearchResults.query` comes from the input string, not from `HybridSearchResults`).

**`singularmem_retrieve::RetrieveOptions` actual fields:**
- `max_blocks: usize` (NOT `limit`)
- `min_score: f32`
- `search: HybridSearchOptions` (NESTED — `fetch_multiplier` and `rrf_k` live here, not at the top level)

**Implication:** the napi `RetrieveOptions { mode, limit, fetchMultiplier, rrfK, minScore }` from the spec needs to map to the nested Rust shape. The handler builds `RetrieveOptions { max_blocks: limit, min_score, search: HybridSearchOptions { limit, fetch_multiplier, rrf_k, include_snippets: false } }`.

**`singularmem_retrieve::MemoryBlock` actual fields** (flat, NOT containing a full Item):
- `id: ItemId`, `content: String`, `score: f32`, `score_kind: ScoreKind`
- `source: Option<String>`, `tags: Vec<String>`, `created_at: Timestamp`
- NO `item` field; NO `supersedes`; NO `metadata`

**Implication:** the spec's `MemoryBlock { item, score, kind }` napi shape does NOT round-trip cleanly. Two options for the implementer:
1. **Match the Rust shape exactly** — change the spec's napi `MemoryBlock` to flat fields: `{ id, content, score, kind, source, tags, createdAt }`. Lose `supersedes` and `metadata` (they're not useful for prompt formatting anyway). This is the simpler choice and matches the Rust intent.
2. **Reconstruct a full Item by fetching from the store** — keep `MemoryBlock { item: Item, score, kind }` but the napi handler does an extra `store.get(block.id)` per block after `Retriever::retrieve` returns. Slower.

**Recommendation:** option 1. The Rust author deliberately omitted `supersedes` and `metadata` from `MemoryBlock` because adapters don't use them. Update the napi `MemoryBlock` to be flat, update the TS interface in `test/types.test.mts`, and update Task 5's `napi_ctx_to_core` to build a flat `singularmem_retrieve::MemoryBlock` directly. This is a real spec change — the spec should be updated post-implementation to reflect what we actually shipped.

**`singularmem_retrieve::RetrievedContext` actual fields:**
- `blocks`, `query`, `elapsed: Duration`, `total_considered: usize`
- `query` IS present (unlike `HybridSearchResults`).

**Implication:** the `RetrievedContext.query` field passes through cleanly. No special handling.

The remaining tasks below assume the implementer applies these corrections inline. Task 1 (foundation) and Task 7 (docs/README) are unaffected; Tasks 2–6 carry intent-correct code that needs field-name and shape adjustments.

// Helper that maps the search crate's errors to the JS code strings.
// Called from both standalone Search-source paths and from the
// unwrapping inside retrieve::Error::Search.
pub(crate) fn map_search_error(e: singularmem_search::Error) -> (&'static str, String) {
    use singularmem_search::Error as SE;
    match e {
        SE::Tantivy { source, .. } => ("Tantivy", format!("Tantivy error: {source}")),
        SE::QueryParse(msg) => ("QueryParse", format!("query parse error: {msg}")),
        SE::IndexMissing { path, .. } => (
            "IndexMissing",
            format!("required index not found at {}", path.display()),
        ),
        SE::IndexCorrupted { path, reason } => (
            "IndexCorrupted",
            format!("index at {} is corrupted: {reason}", path.display()),
        ),
        SE::Io(io) => ("Io", format!("I/O error: {io}")),
        SE::Embedding { source, .. } => ("Embedding", format!("embedder error: {source}")),
        SE::ModelDownload { source, .. } => ("ModelDownload", format!("model download failed: {source}")),
        SE::InvalidModelFiles { reason, .. } => ("InvalidModelFiles", format!("invalid model files: {reason}")),
        SE::DimMismatch { expected, actual } => (
            "DimMismatch",
            format!("embedding dimension mismatch: expected {expected}, got {actual}"),
        ),
        SE::ModelMismatch { expected, found } => (
            "ModelMismatch",
            format!("model mismatch: index built with {expected}, current model {found}"),
        ),
        SE::Usearch { source, .. } => ("Usearch", format!("USearch error: {source}")),
        SE::NoIndexes => (
            "NoIndexes",
            "no search indexes found at sidecar paths; run `singularmem reindex` first".to_string(),
        ),
        SE::HybridMissingIndex { missing, path } => (
            "HybridMissingIndex",
            format!("hybrid mode requested but {missing} sidecar missing at {}", path.display()),
        ),
    }
}
```

- [ ] **Step 6: Add a helper to extract a (code, message) from a `NodeError` source**

This task adds support for wrapping search errors and unwrapping `retrieve::Error::Search`/`Core`. Add a new top-level helper in `src/error.rs`:

```rust
/// Build a NodeError wrapping any source-tagged error from singularmem-search.
pub(crate) fn from_search_error(e: singularmem_search::Error) -> NapiError<&'static str> {
    let (code, message) = map_search_error(e);
    NapiError::new(code, message)
}

/// Build a NodeError wrapping a singularmem-retrieve error.
/// Unwraps Search(_) and Core(_) so the JS error carries the innermost .code.
pub(crate) fn from_retrieve_error(e: singularmem_retrieve::Error) -> NapiError<&'static str> {
    use singularmem_retrieve::Error as RE;
    match e {
        RE::EmptyQuery => NapiError::new("EmptyQuery", "query is empty".to_string()),
        RE::Search(inner) => from_search_error(inner),
        RE::Core(inner) => {
            // Reuse the existing core-error path by going through NodeError -> Error<&'static str>
            NodeError::from(inner).into()
        }
    }
}
```

impl Task for SearchTask {
    type Output = (String, Vec<(singularmem_search::HybridHit, singularmem_core::Item)>);
    type JsValue = crate::types::SearchResults;

    fn compute(&mut self) -> napi::Result<Self::Output> {
        if self.pre_error.is_some() {
            return Err(NapiError::new(napi::Status::GenericFailure, "pre-validation failed"));
        }
        match run_search(
            &self.store,
            &self.store_path,
            &self.query,
            self.mode,
            self.limit,
            self.fetch_multiplier,
            self.rrf_k,

            Ok(out) => Ok(out),
            Err(coded) => {
                self.failed = Some(coded);
                Err(NapiError::new(napi::Status::GenericFailure, "search failed"))
            }
        }
    }

    fn resolve(&mut self, _env: Env, output: Self::Output) -> napi::Result<Self::JsValue> {
        let (query, pairs) = output;
        let hits = pairs
            .into_iter()
            .map(|(hit, item)| crate::types::SearchHit::from_parts(hit, item))
            .collect();
        Ok(crate::types::SearchResults { query, hits })
    }

    fn reject(&mut self, env: Env, _trigger: NapiError) -> napi::Result<Self::JsValue> {
        if let Some(coded) = self.pre_error.take() {
            return Err(coded_error_to_napi_raw(env, coded));
        }
        if let Some(coded) = self.failed.take() {
            return Err(coded_error_to_napi_raw(env, coded));
        }
        Err(NapiError::new(napi::Status::GenericFailure, "unknown search error"))
    }
}

/// Open the sidecars based on what exists, resolve mode, run hybrid search.
/// Returns (echo of query, vec of (hit, fetched item)) so resolve() can build
/// the napi SearchHit. Errors are coded NapiError<&'static str>.
fn run_search(
    store: &Arc<CoreStore>,
    store_path: &std::path::Path,
    query: &str,
    mode: singularmem_search::SearchMode,
    limit: usize,
    fetch_multiplier: usize,
    rrf_k: u32,
) -> Result<(String, Vec<(singularmem_search::HybridHit, singularmem_core::Item)>), NapiError<&'static str>> {
    let tantivy_path = store_path.with_extension("tantivy");
    let vectors_path = store_path.with_extension("vectors");

    let tantivy = if tantivy_path.exists() {
        Some(singularmem_search::Index::open(&tantivy_path).map_err(crate::error::from_search_error)?)
    } else {
        None
    };
    let vectors = if vectors_path.exists() {
        Some(singularmem_search::EmbedderIndex::open(&vectors_path).map_err(crate::error::from_search_error)?)
    } else {
        None
    };

    let searcher = match mode {
        singularmem_search::SearchMode::Lexical => {
            singularmem_search::HybridSearcher::lexical_only(
                tantivy.as_ref().ok_or_else(|| crate::error::from_search_error(
                    singularmem_search::Error::IndexMissing { path: tantivy_path.clone() }
                ))?,
            )
        }
        singularmem_search::SearchMode::Semantic => {
            singularmem_search::HybridSearcher::semantic_only(
                vectors.as_ref().ok_or_else(|| crate::error::from_search_error(
                    singularmem_search::Error::IndexMissing { path: vectors_path.clone() }
                ))?,
            )
        }
        singularmem_search::SearchMode::Hybrid => {
            let t = tantivy.as_ref().ok_or_else(|| crate::error::from_search_error(
                singularmem_search::Error::HybridMissingIndex { missing: "tantivy", path: tantivy_path.clone() }
            ))?;
            let v = vectors.as_ref().ok_or_else(|| crate::error::from_search_error(
                singularmem_search::Error::HybridMissingIndex { missing: "vectors", path: vectors_path.clone() }
            ))?;
            singularmem_search::HybridSearcher::new(t, v)
        }
        singularmem_search::SearchMode::Auto => {
            match (tantivy.as_ref(), vectors.as_ref()) {
                (Some(t), Some(v)) => singularmem_search::HybridSearcher::new(t, v),
                (Some(t), None) => singularmem_search::HybridSearcher::lexical_only(t),
                (None, Some(v)) => singularmem_search::HybridSearcher::semantic_only(v),
                (None, None) => return Err(crate::error::from_search_error(singularmem_search::Error::NoIndexes)),
            }
        }
    };

    let opts = singularmem_search::HybridSearchOptions {
        mode,
        limit,
        fetch_multiplier,
        rrf_k,
    };
    let results = searcher.search(query, &opts).map_err(crate::error::from_search_error)?;

    let mut pairs = Vec::with_capacity(results.hits.len());
    for hit in results.hits {
        let item = store.get(hit.item_id).map_err(|e| {
            crate::error::NodeError::from(e).into()
        })?;
        pairs.push((hit, item));
    }
    Ok((results.query, pairs))
}

/**
 * Pre-prime Tantivy + USearch sidecars, then seed the store. Required for any
 * test that exercises `store.search()` or `store.retrieve()`. The reindex
 * step creates the empty sidecars, then seedStore (5a helper) auto-wires them.

export function seedStoreWithIndexes(path, items) {
  const reindex = spawnSync(
    'cargo',
    ['run', '-q', '-p', 'singularmem', '--', 'reindex', '--with-embeddings', '--store', path],
    { stdio: 'pipe', encoding: 'utf8' },

  if (reindex.status !== 0) {
    throw new Error(
      `reindex failed (exit ${reindex.status}):\nstdout: ${reindex.stdout}\nstderr: ${reindex.stderr}`,
    );
  }
  seedStore(path, items);
}

/// Retrieve a structured context for the given query.
///
/// Runs hybrid search, fetches the full Item for each hit, drops blocks
/// below `minScore`, and returns the resulting `RetrievedContext`. The
/// returned object can be passed to any of the `adapters.{plain,claude,
/// openai,gemini}.format(ctx)` formatters.
///
/// @param query The natural-language query. Must be non-empty.
/// @param options Optional `{ mode?, limit?, fetchMultiplier?, rrfK?, minScore? }`.
/// @returns `RetrievedContext` with `query` echoed and `blocks` populated.
/// @throws `Error` with `.code === "EmptyQuery"` if the query is empty.
/// @throws Same `.code` set as `Store.search` for index-related issues.
#[napi]
#[allow(clippy::missing_errors_doc)]
pub fn retrieve(
    &self,
    query: String,
    options: Option<RetrieveOptions>,
) -> napi::Result<AsyncTask<RetrieveTask>> {
    use singularmem_search::SearchMode;

    let mode_str = options.as_ref().and_then(|o| o.mode.clone()).unwrap_or_else(|| "auto".to_string());
    let mode = match mode_str.as_str() {
        "auto" => SearchMode::Auto,
        "lexical" => SearchMode::Lexical,
        "semantic" => SearchMode::Semantic,
        "hybrid" => SearchMode::Hybrid,
        other => {
            let coded = NapiError::new(
                "Validation",
                format!("unknown search mode '{other}'; expected one of: auto, lexical, semantic, hybrid"),
            );
            return Ok(AsyncTask::new(RetrieveTask {
                store: self.inner.clone(),
                store_path: self.path.clone(),
                query,
                mode: SearchMode::Auto,
                limit: 10,
                fetch_multiplier: 3,
                rrf_k: 60,
                min_score: 0.0,
                pre_error: Some(coded),
                failed: None,
            }));
        }
    };

    #[allow(clippy::cast_possible_truncation)]
    let limit = options.as_ref().and_then(|o| o.limit).unwrap_or(10) as usize;
    #[allow(clippy::cast_possible_truncation)]
    let fetch_multiplier = options.as_ref().and_then(|o| o.fetch_multiplier).unwrap_or(3) as usize;
    let rrf_k = options.as_ref().and_then(|o| o.rrf_k).unwrap_or(60);
    #[allow(clippy::cast_possible_truncation)]
    let min_score = options.and_then(|o| o.min_score).unwrap_or(0.0) as f32;

    Ok(AsyncTask::new(RetrieveTask {
        store: self.inner.clone(),
        store_path: self.path.clone(),
        query,
        mode,
        limit,
        fetch_multiplier,
        rrf_k,

    }))

/// Convert the napi-exposed RetrievedContext back into the core type so an
/// Adapter's format() can consume it. The napi shape is sent from JS, so we
/// need to reconstruct ItemIds (parse from string) and rebuild MemoryBlocks.
fn napi_ctx_to_core(ctx: crate::types::RetrievedContext) -> singularmem_retrieve::RetrievedContext {
    use std::str::FromStr;
    let blocks = ctx
        .blocks
        .into_iter()
        .map(|b| {
            let item = singularmem_core::Item {
                id: singularmem_core::item::ItemId::from_str(&b.item.id)
                    .unwrap_or_else(|_| singularmem_core::item::ItemId::new()),
                content: b.item.content,
                #[allow(clippy::cast_possible_truncation, clippy::cast_sign_loss)]
                created_at: jiff::Timestamp::from_millisecond(b.item.created_at as i64).unwrap_or_else(|_| jiff::Timestamp::now()),
                supersedes: b.item.supersedes.and_then(|s| singularmem_core::item::ItemId::from_str(&s).ok()),
                tags: b.item.tags,
                source: b.item.source,
                metadata: b.item.metadata,
            };
            singularmem_retrieve::MemoryBlock {
                item,
                #[allow(clippy::cast_possible_truncation)]
                score: b.score as f32,
                kind: match b.kind.as_str() {
                    "rrf" => singularmem_search::ScoreKind::Rrf,
                    "bm25" => singularmem_search::ScoreKind::Bm25,
                    "cosine" => singularmem_search::ScoreKind::Cosine,
                    _ => singularmem_search::ScoreKind::Rrf,  // shouldn't happen given our own JS shape
                },
            }
        })
        .collect();
    singularmem_retrieve::RetrievedContext {
        query: ctx.query,
        blocks,
    }
}

## Search

Run a hybrid search over the store's indexes (Tantivy lexical + USearch
semantic + RRF fusion).

```javascript
import { Store } from 'singularmem';

const store = await Store.open('./memory.db');

const results = await store.search('cat care', {
  mode: 'hybrid',     // 'auto' (default) | 'lexical' | 'semantic' | 'hybrid'
  limit: 10,          // default 10
  fetchMultiplier: 3, // default 3
  rrfK: 60,           // default 60
});

for (const hit of results.hits) {
  console.log(hit.score, hit.kind, hit.item.content);
}
```

Indexes must exist on disk before `search()` can find anything. Build
them via the CLI: `singularmem reindex --with-embeddings --store ./memory.db`.

Mode `'auto'` probes for what's available and degrades. Explicit modes
fail with `code: 'IndexMissing'` or `code: 'HybridMissingIndex'` if a
required sidecar is absent.

## Retrieve

Higher-level convenience that runs search, fetches the full Item for each
hit, and returns a structured `RetrievedContext` suitable for passing to
an adapter.

```javascript
import { Store, adapters } from 'singularmem';

const store = await Store.open('./memory.db');
const ctx = await store.retrieve('cat care', {
  minScore: 0.1,  // filter low-scoring blocks
  limit: 5,
});

// Format for whichever LLM provider you're calling:
const prompt = adapters.claude.format(ctx);
```

## Adapters

Four pre-built adapters cover the constitutional Principle II providers:

- `adapters.plain` — Markdown blocks with `## memory N` headings
- `adapters.claude` — Anthropic `<documents><document index="N">` XML
- `adapters.openai` — Bracketed `[N]` citations with leading instruction
- `adapters.gemini` — Em-dash `Source N` headers with grounding directive

Each exposes a `name` property and a synchronous `format(ctx)` method:

```javascript
adapters.claude.format(ctx);  // returns string
```

Custom JS adapters are not supported in this release — if you need a
different format, build the string yourself from the `RetrievedContext`
that `store.retrieve()` returns.

## Error handling (5b additions)

In addition to the 5a error codes, search and retrieve can throw:

| Code | Meaning |
|---|---|
| `NoIndexes` | `mode: 'auto'` but no sidecar indexes exist |
| `IndexMissing` | Explicit mode requires a sidecar that's absent |
| `HybridMissingIndex` | `mode: 'hybrid'` but one of the two sidecars is missing |
| `EmptyQuery` | `store.retrieve('')` rejects with this (search returns empty hits instead) |
| `QueryParse` | Tantivy query syntax error |
| `Tantivy` | Tantivy-specific runtime error |
| `Usearch` | USearch-specific runtime error |
| `Embedding` | Embedder runtime error |
| `ModelDownload` | fastembed model download failure |
| `InvalidModelFiles` | Embedder model files malformed |
| `DimMismatch` | Vector dimension mismatch |
| `ModelMismatch` | Sidecar built with a different embedder model |
```
