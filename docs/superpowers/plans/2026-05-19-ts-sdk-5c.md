**Goal:** Add `store.ingest(item)` to the `singularmem-node` napi binding, closing the read-write gap in the JS SDK. Auto-wire Tantivy + USearch hooks when sidecars exist (transparent to JS callers).

**Tech Stack:** Unchanged from 5a/5b — napi 2, tokio rt-multi-thread, fastembed via env-var-switchable embedder selection, MockEmbedder for tests.

- [ ] **Step 2: Add `js_new_item_to_core` helper at the bottom of `src/types.rs`**

```rust
/// Convert a JS-sent `NewItem` into the core's `NewItem`. Performs early
/// validation of the `supersedes` ULID format; returns a coded NapiError
/// (with `.code === "InvalidId"`) if the string isn't a valid ULID.
/// Empty-string `supersedes: ""` is normalized to `None` (defensive: TS
/// callers might pass empty strings from form fields).
///
/// # Errors
///
/// Returns a `NapiError<&'static str>` with `.code === "InvalidId"` if
/// `supersedes` is set to a malformed ULID string.
pub fn js_new_item_to_core(
    item: NewItem,
) -> Result<singularmem_core::NewItem, napi::Error<&'static str>> {
    use std::str::FromStr;

    let supersedes = match item.supersedes.as_deref() {
        Some(s) if !s.is_empty() => Some(
            singularmem_core::item::ItemId::from_str(s).map_err(|e| {
                let core_err = singularmem_core::Error::from(e);
                crate::error::NodeError::from(core_err).into()
            })?,
        ),
        _ => None,
    };

    Ok(singularmem_core::NewItem {
        content: item.content,
        supersedes,
        tags: item.tags.unwrap_or_default(),
        source: item.source,
        metadata: item.metadata.unwrap_or_else(|| serde_json::json!({})),
    })
}
```

## Task 2: Refactor `open_sidecars` to non-fatal semantics

Change the existing 5b helper so individual `Index::open` / `EmbedderIndex::open` failures log + return `None` for the failing sidecar instead of bubbling. Benefits search/retrieve (graceful degradation when one sidecar is corrupted) AND enables 5c's auto-wiring to use the same helper.

**Files:**
- Modify: `crates/singularmem-node/src/store.rs`

- [ ] **Step 1: Read the current `open_sidecars` implementation**

Run: `grep -n "fn open_sidecars" crates/singularmem-node/src/store.rs` to find the function.

The current shape returns `Result<(Option<Index>, Option<EmbedderIndex>), NapiError<&'static str>>` — the outer `Result` is `Err` when an open fails. The change: keep the outer `Result` for I/O errors at the sidecar-existence check (e.g., permission denied probing the path), but treat individual `Index::open` failures as inner-`None` returns with a `tracing::warn!` instead.

- [ ] **Step 2: Apply the refactor**

Change the function body so the structure is:

```rust
fn open_sidecars(
    store_path: &std::path::Path,
) -> Result<
    (
        Option<singularmem_search::Index>,
        Option<singularmem_search::EmbedderIndex>,
    ),
    napi::Error<&'static str>,
> {
    let tantivy_path = {
        let mut p = std::ffi::OsString::from(store_path);
        p.push(".tantivy");
        std::path::PathBuf::from(p)
    };
    let vectors_path = {
        let mut p = std::ffi::OsString::from(store_path);
        p.push(".vectors");
        std::path::PathBuf::from(p)
    };

    let tantivy = if tantivy_path.exists() {
        match singularmem_search::Index::open(&tantivy_path) {
            Ok(idx) => Some(idx),
            Err(e) => {
                tracing::warn!(
                    path = %tantivy_path.display(),
                    error = %e,
                    "Tantivy sidecar present but failed to open; skipping"
                );
                None
            }
        }
    } else {
        None
    };

    let vectors = if vectors_path.exists() {
        let embedder: Box<dyn singularmem_search::Embedder> =
            match std::env::var("SINGULARMEM_TEST_EMBEDDER").ok().as_deref() {
                Some("mock") => Box::new(singularmem_search::testing::MockEmbedder::default()),
                _ => match singularmem_search::FastembedEmbedder::new() {
                    Ok(e) => Box::new(e),
                    Err(e) => {
                        tracing::warn!(error = %e, "FastembedEmbedder::new() failed; vectors sidecar will be skipped");
                        return Ok((tantivy, None));
                    }
                },
            };
        match singularmem_search::EmbedderIndex::open(&vectors_path, embedder) {
            Ok(idx) => Some(idx),
            Err(e) => {
                tracing::warn!(
                    path = %vectors_path.display(),
                    error = %e,
                    "Vectors sidecar present but failed to open; skipping"
                );
                None
            }
        }
    } else {
        None
    };

    Ok((tantivy, vectors))
}
```

Notes:
- The outer `Result` is now reserved for catastrophic errors that prevent even starting probing — but realistically there aren't any left, so the function always returns `Ok(_)`. The `Result` return type is kept for signature stability with existing callers.
- If you find that no callsite ever sees `Err`, simplifying the return type to `(Option<Index>, Option<EmbedderIndex>)` (no Result) is a reasonable cleanup. Up to the implementer.

```rust
fn compute(&mut self) -> napi::Result<Self::Output> {
    if self.pre_error.is_some() {
        return Err(NapiError::new(napi::Status::GenericFailure, "pre-validation failed"));
    }
    let mut store = match CoreStore::open_with_options(
        &self.path,
        CoreStoreOptions { read_only: self.read_only },
    ) {
        Ok(s) => s,
        Err(e) => {
            self.failed = Some(NodeError::from(e));
            return Err(NapiError::new(napi::Status::GenericFailure, "open failed"));
        }
    };

    // Auto-wire hooks ONLY when the store is read-write. Read-only stores
    // reject writes at the SQLite layer; hooks would be unused.
    if !self.read_only {
        // open_sidecars is non-fatal per Task 2; per-sidecar failures
        // already logged. Any catastrophic error is recoverable here too —
        // log it and continue without hooks rather than failing Store.open.
        let (tantivy, vectors) = open_sidecars(&self.path).unwrap_or_else(|e| {
            tracing::warn!(error = ?e, "sidecar probe failed during Store.open; ingest will skip hooks");
            (None, None)
        });

        let mut hooks: Vec<Box<dyn singularmem_core::IndexHook>> = Vec::new();
        if let Some(idx) = tantivy {
            hooks.push(Box::new(idx));
        }
        if let Some(idx) = vectors {
            hooks.push(Box::new(idx));
        }
        if !hooks.is_empty() {
            store.set_hook(Some(Box::new(singularmem_core::hook::MultiHook::new(hooks))));
        }
    }

    Ok(Arc::new(store))
}
```

Verify `singularmem_core::IndexHook` is in scope (add a `use` or qualify). Verify `singularmem_core::hook::MultiHook::new(hooks)` is the correct constructor (check `crates/singularmem-core/src/hook.rs`).

**Important wrinkle:** the existing search/retrieve paths ALSO call `open_sidecars` (per-call probing). Now that `Store.open` also calls it (and potentially opens the same sidecars), there will be two open handles to each sidecar when both ingest and search are exercised in the same JS process. This is acceptable for v0 (acknowledged in the spec as a future optimization). Tantivy and USearch both support multiple concurrent readers.

```rust
// ── IngestTask ───────────────────────────────────────────────────────────────

pub struct IngestTask {
    store: Arc<CoreStore>,
    new_item: Option<singularmem_core::NewItem>,
    pre_error: Option<NapiError<&'static str>>,
    failed: Option<NodeError>,
}

#[napi]
impl Task for IngestTask {
    type Output = singularmem_core::Item;
    type JsValue = crate::types::Item;

    fn compute(&mut self) -> napi::Result<Self::Output> {
        if self.pre_error.is_some() {
            return Err(NapiError::new(napi::Status::GenericFailure, "pre-validation failed"));
        }
        let new_item = self
            .new_item
            .take()
            .expect("new_item must be Some when pre_error is None");
        match self.store.ingest(new_item) {
            Ok(item) => Ok(item),
            Err(e) => {
                self.failed = Some(NodeError::from(e));
                Err(NapiError::new(napi::Status::GenericFailure, "ingest failed"))
            }
        }
    }

    fn resolve(&mut self, _env: Env, output: Self::Output) -> napi::Result<Self::JsValue> {
        Ok(output.into())
    }

    fn reject(&mut self, env: Env, _trigger: NapiError) -> napi::Result<Self::JsValue> {
        if let Some(coded) = self.pre_error.take() {
            return Err(coded_error_to_napi_raw(env, coded));
        }
        let node_err = self.failed.take().unwrap_or_else(|| {
            NodeError::from(singularmem_core::Error::Io(std::io::Error::other(
                "unknown ingest error",
            )))
        });
        Err(node_error_to_napi_with_raw(env, node_err))
    }
}
```

After the existing `retrieve` method:

```rust
/// Persist a new item to the store.
///
/// If the store was opened with Tantivy + USearch sidecars present, the
/// new item will also be written to those indexes (auto-wired hooks).
/// Hook write failures during ingest are logged via tracing::warn! but
/// do NOT roll back the SQLite insert — the returned Item is durable
/// regardless of hook outcomes (Principle VII).
///
/// @param item The item to persist. Only `content` is required.
/// @returns The newly-persisted Item with assigned `id` and `createdAt`.
/// @throws Error with `.code === "Validation"` if any field fails validation
///   (empty content, content > 1 MiB, etc.).
/// @throws Error with `.code === "InvalidId"` if `supersedes` is set to a
///   malformed ULID string.
/// @throws Error with `.code === "SupersedesNotFound"` if `supersedes`
///   references an ID that does not exist in the store.
/// @throws Error with `.code === "ReadOnly"` if the store was opened with
///   `{ readOnly: true }`.
/// @throws Error with `.code === "Sqlite"` if the underlying database
///   write fails.
#[napi]
#[allow(clippy::missing_errors_doc)]
pub fn ingest(&self, item: crate::types::NewItem) -> napi::Result<AsyncTask<IngestTask>> {
    // Validate supersedes ULID synchronously so a malformed string fails
    // fast (still as a rejected Promise via the established pre_error path).
    match crate::types::js_new_item_to_core(item) {
        Ok(new_item) => Ok(AsyncTask::new(IngestTask {
            store: self.inner.clone(),
            new_item: Some(new_item),
            pre_error: None,
            failed: None,
        })),
        Err(coded) => Ok(AsyncTask::new(IngestTask {
            store: self.inner.clone(),
            new_item: None,
            pre_error: Some(coded),
            failed: None,
        })),
    }
}
```

```javascript
// Tests for hook auto-wiring: requires pre-primed Tantivy + USearch sidecars.
process.env.SINGULARMEM_TEST_EMBEDDER = 'mock';

import { test } from 'node:test';
import assert from 'node:assert/strict';
import { spawnSync } from 'node:child_process';
import { Store } from '../index.js';
import { freshStorePath } from './helpers.mjs';

function primeSidecars(path) {
  const r = spawnSync(
    'cargo',
    ['run', '-q', '-p', 'singularmem', '--', 'reindex', '--with-embeddings', '--store', path],
    {
      stdio: 'pipe',
      encoding: 'utf8',
      env: { ...process.env, SINGULARMEM_TEST_EMBEDDER: 'mock' },
    },
  );
  if (r.status !== 0) throw new Error(`reindex failed: ${r.stderr}`);
}

  const { path } = freshStorePath();

  await store.ingest({ content: 'no indexes yet' });
  await assert.rejects(

    (err) => err.code === 'NoIndexes',
  );

test('Store.ingest after pre-priming sidecars makes the item searchable', async () => {
  const { path } = freshStorePath();
  primeSidecars(path);
  const store = await Store.open(path);
  // Exact-match query so MockEmbedder produces a hit
  await store.ingest({ content: 'cats are great pets' });
  const results = await store.search('cats are great pets');
  assert.ok(
    results.hits.length >= 1,
    'newly ingested item should be searchable via the auto-wired hooks',
  );
});
```

```markdown
const v1 = await store.ingest({ content: 'old version' });

with \`code: 'ReadOnly'\`. Hooks are not attached on read-only stores.
```
