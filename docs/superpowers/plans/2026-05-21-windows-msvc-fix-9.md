## Task 1: Create branch + `.cargo/config.toml`

**Files:**
- Create: `.cargo/config.toml`

This is the load-bearing change. The rustflag forces `cxx` (and any cc-rs-compiled C++) to use `/MD` dynamic CRT instead of `/MT` static CRT, matching ort-sys's pre-built onnxruntime objects.

- [ ] **Step 1: Create the sub-project branch**

```bash

git checkout main
git pull --ff-only
git checkout -b windows-msvc-fix-9
```

Verify: `git branch --show-current` returns `windows-msvc-fix-9`.

- [ ] **Step 2: Create `.cargo/config.toml`**

```bash
cat > .cargo/config.toml <<'EOF'
# Cargo build configuration.
#
# `target-feature=-crt-static` for x86_64-pc-windows-msvc:
# Forces the cxx crate (and any cc-rs-compiled C++ in our dep graph) to
# use /MD (dynamic CRT) linkage on Windows MSVC. This matches ort-sys's
# pre-built onnxruntime binaries, which are shipped by Microsoft with
# /MD and cannot be rebuilt. Without this flag, the linker fails with:
#   error LNK2038: mismatch detected for 'RuntimeLibrary':
#     value 'MT_StaticRelease' doesn't match 'MD_DynamicRelease'
#
# See docs/superpowers/specs/2026-05-21-windows-msvc-fix-9-design.md
# for the full investigation.
[target.x86_64-pc-windows-msvc]
rustflags = ["-C", "target-feature=-crt-static"]
EOF
```

- [ ] **Step 3: Verify the file parses as valid TOML**

```bash
python3 -c "import tomllib; tomllib.load(open('.cargo/config.toml', 'rb'))" && echo "valid TOML"
```

Expected: `valid TOML`.

If `tomllib` isn't available (Python < 3.11), substitute `python3 -c "import tomli; tomli.load(open('.cargo/config.toml', 'rb'))"` after `pip install tomli`, OR check with `cargo --version` (cargo will reject the file on the next `cargo` invocation if it's malformed).

- [ ] **Step 4: Sanity check — non-Windows build still works**

```bash
cargo check --workspace 2>&1 | tail -3
```

Expected: completes successfully on the current macOS/Linux host. The `[target.x86_64-pc-windows-msvc]` table is ignored for other targets.

- [ ] **Step 5: Commit**

```bash
git add .cargo/config.toml
git commit -s -m "fix(build): force -crt-static off for x86_64-pc-windows-msvc

The cxx crate (and other cc-rs-compiled C++ in our dep graph) compiles
with /MT (static CRT) by default on Windows MSVC, while ort-sys ships
pre-built onnxruntime binaries from Microsoft that use /MD (dynamic
CRT). The linker rejects mixing /MT and /MD with error LNK2038.

Setting target-feature=-crt-static via .cargo/config.toml forces cxx
to use /MD, matching ort-sys. Only affects builds targeting
x86_64-pc-windows-msvc; other targets are unaffected.

This unblocks the Windows MSVC binary in cargo-dist's release matrix,
which was dropped during the v0.15.0 saga (sub-project 7).

Sub-project 9."
```
